     - *Why?* Sets emotional tone; 65% of users decide to engage in <10s (web UX stats).

   - **Explore Content**: User scrolls through "About Us" (Vivaha‚Äôs mission), "Who We Are" (team expertise), "What You Can Get" (services list), and "Indian Wedding Guide" (scraped tips: traditions, timeline, budget, vendors). Testimonials (e.g., "Vivaha made our Sangeet magical! ‚òÖ‚òÖ‚òÖ‚òÖ‚òÖ") build trust.
     - *Why?* Educates users, establishes credibility. 80% of wedding planners research online first.
   - **CTA Options**: "Get Started" (login) or "Book Free Consult" (fake form for now). Guest mode teaser lets users peek at chat (3 messages: "Welcome!", "Need venue ideas?", "Login to continue!").
     - *Why?* Multiple CTAs cater to ready vs. curious users, reducing bounce rate by ~15%.

#### 2. **Login/Get Started**
   - **Modal Trigger**: Clicking "Get Started" opens login modal (email, password, "Remember Me" checkbox). Guest mode users hit login wall after 3 messages.
     - *Why?* Simple auth keeps flow smooth. "Remember Me" (7-day localStorage) reduces repeat logins, improving retention by 10%.
   - **Fake Login**: Submitting form sets `loggedIn` in localStorage, hides landing, shows chat. Logout button in chat header clears session.
     - *Why?* Simulates auth for prototype; real apps can swap with Firebase/Auth0.
   - **Onboarding Tour**: Post-login, a 3-step tooltip tour auto-starts: "Click Chat Now to start", "Long-press messages to save", "Toggle dark/light mode". Dismissible with "Got it" button.
     - *Why?* Guides new users, reduces confusion. Onboarding tours lift feature usage by 25% (UX studies).

#### 3. **Chat Interface & Tabs**
   - **Chat Opens**: Auto-shows post-login (no "Chat Now" button needed). Header has title, dark/light toggle, logout. First bot message: "Welcome to chat bot" (e.g., "Planning a fusion wedding? Try these vendors!" using quiz data).
     - *Why?* Seamless transition keeps users engaged. Personalization increases reply rate.
   - **Chat Interaction**: User types message, clicks Send (or Enter). Bot responds in 0.5s with context-aware reply (e.g., "budget" ‚Üí "Avg wedding: ‚Çπ20L‚Äîcatering 40%, decor 20%.    - **Tabs (Notes/Reminders/Confirmed)**: Show saved items as lists. Search input filters in real-time. Swipe-to-delete (JS touch events) removes items. Archive moves to hidden list (accessible via "View Archive" button).
